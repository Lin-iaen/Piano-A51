1. Lib -- 纯逻辑层 —— 熟悉基本指令与堆栈
    重点在函数开头的 PUSH 和结尾的 POP。在汇编里，寄存器 R0-R7 是公共财产 ，在函数里要用它们，就必须先 PUSH 存进堆栈（压入），用完后再 POP 还回去（弹出），而且顺序必须是先进后出（镜像对称）

2. Drv 裸机驱动层 —— 掌控硬件寄存器与中断
    SFR (特殊功能寄存器) 操作： 重点在于怎么用 ANL (与)、ORL (或) 来配置 TMOD 和 TCON
    看 UART_ISR 和 Timer0_ISR。重点理解为什么中断函数以 RETI 结尾（而不是 RET）。学习环形缓冲区是怎么用汇编里的间接寻址 MOV @R0, A 来存数据的

3. Bsp 板级支持包 —— 查表与硬件映射
    重点在 MOVC A, @A+DPTR 这条指令。这是 51 单片机唯一从 ROM（Flash）里读取表格数据的方法。低、中、高音频率数据靠它取出来

    条件判断： 关注 JZ (如果是 0 就跳转) 和 SUBB (带借位减法) 是怎么配合实现类似 C 语言里 if-else 的。
4. App 业务逻辑层 —— 状态机与整体调度
    多重分支： 汇编里没有 switch-case，全靠 CJNE (比较不相等则跳转) 和 JC (如果有借位/小于则跳转) 来判断用户按下了哪个键

    链接与总控： 看 Main.asm 是如何把分散的文件用 EXTRN 和 LCALL 串联起来，并妥善安放中断向量表（ORG 000BH 等）

5. 2026-04 扩展实现约束（第一阶段）
    架构变更：输入源（串口/按键/红外）统一输出“事件码”，由 Main.asm 中统一分发器执行 Buzzer_Play 和 Led_ShowSpectrum。
    红外接入：新增 Drv_IR.asm + Protocol_IR.asm，INT0(0003H) 挂接红外中断，按 NEC 常见 21 键遥控映射。
    引脚约束：P3.2 现在保留给 INT0 红外输入，按键扫描位改为 P3.3/P3.4/P3.5（三键）。
    维护建议：后续新增输入方式时，不要在协议/驱动层直接调用 Buzzer_Play，统一通过事件分发器进入业务层。

6. 下一阶段与上位机需求
    下一阶段执行重点：
    1) 完成红外事件码到主循环分发器的全链路联调，保证串口/按键/红外三路输入统一行为。
    2) 建立协议一致性测试用例，覆盖正常帧、丢帧、乱序、重复帧与超时重传场景。
    3) 补齐运行期可观测数据（收发计数、错误码、重传次数、心跳状态），便于快速定位现场问题。
    4) 形成版本化协议文档与联调基线，确保固件与上位机可独立迭代。

    上位机职责定位：
    上位机从“发音符工具”升级为“系统总控中心”，负责连接管理、指令编排、状态监控、数据留存与调试支撑；单片机侧聚焦实时执行与硬件闭环。

    上位机功能需求：
    - 设备连接管理：串口自动发现、连接/断开、参数协商、在线状态监测。
    - 实时控制：音符下发、播放/暂停/停止、模式切换、批量命令与优先级控制。
    - 曲谱/录制管理：曲谱导入导出、录制回放、版本与标签管理。
    - 日志与调试：收发报文查看、错误码解释、统计报表、问题回放。
    - 参数配置：节拍、音量映射、按键映射、协议版本与超时参数配置。

    通信协议建议：
    - 帧格式：帧头(2B, 0x55AA) + 版本(1B) + 类型(1B) + 长度(2B) + 载荷(NB) + 校验(2B, CRC16)。
    - ACK/重传：命令帧要求ACK，超时未确认则按次数上限重传；重复包用序号去重。
    - 心跳：双向周期心跳，连续丢失阈值后进入降级或重连流程，并上报离线事件。

    UI建议：
    - 核心页面：设备连接页、实时控制台、曲谱/录制页、日志诊断页、参数配置页。
    - 状态可视化：连接质量、延迟趋势、缓冲区占用、错误率、心跳与重传计数的实时图表。

    非功能需求：
    - 延迟：端到端控制延迟目标小于50ms，抖动可控。
    - 稳定性：长时间运行无内存泄漏或死锁，异常自动恢复。
    - 可扩展性：协议字段预留，支持新增命令类型与多设备管理。
    - 可观测性：关键路径全量埋点，支持按时间线追踪与问题复盘。
