============================================================
 PC_piano 项目说明 (小白入门版)
============================================================

【这个项目是什么？】

  一个用 8051 单片机做的电子琴。你可以通过三种方式让它发声：
    1. 在电脑上通过串口发命令（就像用键盘弹琴）
    2. 用红外遥控器按键（就像用电视遥控器控制）
    3. 按板子上的物理按钮
  它会把声音通过蜂鸣器放出来，同时用 8 个 LED 灯显示当前音符的高低。
  还内置了几首自动播放的歌曲（两只老虎、小星星、欢乐颂、生日快乐）。

【项目目录结构是怎么组织的？（从下到上越靠近硬件）】

  Lib/     基础库 — 提供延时、数学运算等最底层的工具函数
  Drv/     驱动层 — 直接操作单片机硬件（定时器、串口、红外接收头）
  Bsp/     板级支持层 — 把硬件操作包装成"蜂鸣器发声""LED亮"等人能懂的接口
  App/     应用层 — 协议解析、歌曲播放、事件分发等业务逻辑
  Readme/  文档 — 就是你现在看的这个

  调用关系永远是从上往下的：App 调 Bsp 调 Drv 调 Lib。
  App 之间互相不调，Bsp 之间也互相不调，层次清晰。

【怎么工作的？一张图看懂】

  串口(PC) ─┐
  红外遥控  ─┼──→ 各自翻译成"统一事件码" ──→ Dispatch_Event ─┬──→ 蜂鸣器发声
  物理按键  ─┘                                            └──→ LED频谱显示

  事件码对照表：
    0      → 什么都不做
    1~21   → 播放单个音符（C3到B5共21个半音）
    22     → 静音
    30~39  → 播放歌曲（歌曲ID为0~9）

【想加新功能怎么办？】

  加一个新输入源（比如蓝牙、WiFi？）：
    1. 写一个类似 Protocol.asm 的文件，把新输入的数据翻译成事件码
    2. 在 Main.asm 主循环里加一行 LCALL 你的新解析函数
    3. 不用改 Dispatch_Event，它不认识的事件码直接忽略

  加一首新歌：
    1. 在 MusicPlayer.asm 底部仿照现有格式写歌曲数据
    2. 在 SongTable 里加一个 DW 指向你的新歌
    3. 注意 SongID 不要超过 9（当前只有 10 个槽位）

【怎么编译？】

  用 Keil C51 (uVision) 打开 PC_piano.uvproj → 点 Build。
  生成 PC_piano.hex 文件，烧录到芯片即可。

【维护注意事项】

  - 中断和主循环共享的数据（如 UART_RxBuf）要注意"原子读取"：
    关中断 → 复制 → 开中断，防止读到一半被新数据覆盖。
  - 不直接调 Buzzer_Play，而是通过 Dispatch_Event 走统一事件系统。
  - 改串口协议后，PC上位机和单片机代码要同步更新。

【文件清单（按推荐阅读顺序）】

  纯小白建议按这个顺序看代码，由浅入深：
    1. Lib/Delay.asm     — 最简单的延时函数，指令少，好懂
    2. Bsp/Bsp_Led.asm   — 查表输出LED，理解 MOVC 指令
    3. Drv/Drv_Timer.asm — 定时器中断，理解方波怎么产生的
    4. Bsp/Bsp_Buzzer.asm — 音符查表发声，理解频率怎么控制
    5. Bsp/Bsp_Key.asm   — 按键消抖状态机，理解"等15ms确认"原理
    6. App/Protocol.asm  — 串口数据解析，理解"原子快照"防并发
    7. Drv/Drv_Uart.asm  — 最复杂的文件，8状态帧接收机
    8. Drv/Drv_IR.asm    — 红外NEC协议解码，理解间隔测量
    9. App/Protocol_IR.asm — 遥控码翻译表
   10. App/MusicPlayer.asm — 歌曲自动播放引擎
   11. App/Main.asm      — 总入口，看完前面的再来看这个就全通了
