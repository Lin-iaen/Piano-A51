============================================================
 PC 上位机说明 (小白版)
============================================================

【上位机是什么？】

  上位机就是运行在普通电脑(Windows/Linux/Mac)上的一个程序，
  它通过 USB转串口线 连接到单片机，给单片机发送命令。
  
  在这个项目里，上位机的角色是"远程琴键"——
  你在电脑上点一个按钮，电脑通过串口告诉单片机"弹 Do 音"，
  单片机收到后就驱动蜂鸣器发出 Do 的声音。

【上位机和单片机怎么分工？】

  上位机(PC)：                   单片机(8051)：
  ┌─────────────────┐           ┌─────────────────┐
  │ 界面 + 用户交互   │  串口线   │ 实时执行         │
  │ 发送音符 / 命令   │─────────→│ 驱动蜂鸣器       │
  │ 显示日志 / 状态   │←─────────│ (暂不回传状态)    │
  └─────────────────┘           └─────────────────┘

【串口通信协议（上位机必须遵守的"语言规则"）】

  数据的"包裹"格式（每一帧）：
    ┌──────┬──────┬──────┬──────┬──────┬───────────┬────────┐
    │ 0x55 │ 0xAA │ 0x01 │ Cmd  │ Len  │ Payload[] │ CRC16  │
    │ 帧头1 │帧头2 │ 版本 │ 命令 │ 长度 │  荷载数据  │ 校验码 │
    └──────┴──────┴──────┴──────┴──────┴───────────┴────────┘

  当前支持的命令：
    Cmd = 0x02  (音符控制)
      Payload[0] = 0    → 静音
      Payload[0] = 1~21 → 弹奏对应音符 (C3~B5)
      Payload[0] >= 22  → 非法值，单片机忽略

  参数配置：
    波特率：9600
    数据位：8
    停止位：1
    校验位：无 (No Parity)

【上位机开发的建议】

  最小功能：
    - 能选择串口号并打开连接
    - 有 22 个按钮（静音 + 21个音符），按一下就发一帧
    - 显示发送和接收的原始报文日志
    - 连接断开时提示用户

  进阶功能：
    - 按键按下时持续发送（按住Do=持续发声），松开发送静音帧
    - 记录丢帧/错误帧统计
    - 虚拟钢琴键盘界面（而不是纯按钮）

【联调要点】

  - 串口参数必须和单片机一致（9600-8-N-1）
  - 发送前确保 Payload[0] 在 0~21 合法范围
  - 如果连续发帧，每帧之间留几毫秒间隔（单片机8051速度有限）
  - 长时间测试时观察是否有漏音、卡音
